# Product

## Register

product

## Users
ryOS primarily serves creative tinkerers and retro fans. They use it to explore a nostalgic desktop environment, play with built-in apps, and create things inside the system rather than just consume static content. The core jobs to be done are exploring nostalgia and creating or playing within a polished browser-based OS.

### Brand Personality
The brand should feel nostalgic, playful, inventive, and crafted. The interface should balance delight, confidence, calm, curiosity, and a light sense of energy. It should feel intentional and characterful, not generic or purely utilitarian.

### Aesthetic Direction
Favor a balanced direction: preserve the retro soul of the product while applying modern usability polish. Optimize visual decisions macOS-first, while keeping the experience coherent across all shipped themes. Stay close to the current theme palettes, typography, iconography, and OS-specific visual language already present in the repo.

### Design Principles
1. Preserve retro identity, then refine usability. Keep the period-specific character intact, but remove friction where modern polish clearly helps.
2. Design macOS-first, validate across all themes. Aqua is the primary showcase, but new work must still hold up in System 7, XP, and Windows 98.
3. Reuse the shipped visual system. Prefer existing `--os-*` tokens, fonts, icons, and palettes over inventing new stylistic directions.
4. Make play feel trustworthy. The UI should be exploratory and delightful without sacrificing clarity, stability, or confidence.
5. Treat accessibility as a product requirement. Target WCAG AA, preserve reduced-motion support, ensure keyboard accessibility, and never rely on color alone.
